# Forge Project Vision & Philosophy

This document outlines the long-term vision, core philosophy, and scope of Forge.

---

## 1. The Mission

To build a premium, open-source visual scripting workspace that empowers game designers and developers to design narrative logic, state machines, quest paths, and AI behaviors seamlessly, without lock-in to proprietary editor architectures.

---

## 2. Core Philosophy

### visual logic, runtime decoupled
Forge is a visual drafting board, not a runtime engine. It compiles schemas but does not execute them in-game. Game runtime execution is left entirely to the game engine loaders, keeping Forge lightweight, clean, and universal.

### The "No Black Box" Standard
All file formats generated by Forge (`.forge`) are fully human-readable and trackable under Git. Merges, diffs, and inspects of visual project state must be as clean as inspecting standard source code.

---

## 3. Project Status & Vision Scope

| Vision Layer | Status | Target Description |
|---|---|---|
| **Core Framework** | `Implemented` | Node Registry, Inspector sidebars, and dynamic handles. |
| **Workspace Basics** | `Implemented` | Autosave, History queueing, notifications, settings, and local project serialization. |
| **File Systems** | `Planned` | Native file loading (`.forge` parser) and desktop file triggers. |
| **Asset Library** | `Future Vision` | Asset indexing pipeline mapping game assets (sprites, sound files) directly onto node dropdown selectors. |
